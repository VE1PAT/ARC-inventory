ARC Inventory — Locked Design Spec

Halifax Amateur Radio Club · halifax-arc.org · Repo: VE1PAT/ARC-inventory

Status: approved for implementation · Callsign-only · Witness-based loans · Online only

1. Purpose

Web inventory for club assets (hardware, software, books, kits, fixed station gear, etc.). Members loan and return gear with a second-member Witness. Accountability is primary: append-only ledger, no hard deletes, callsigns only (no personal data).

2. Roles

RoleWhoCapabilities
Superuser Multiple people (no single point of failure) Everything Admin can do, plus manage admins/superusers and system settings. Cannot remove the last superuser. Warn if fewer than 2 active superusers.
Admin Executives and assigned members (e.g. membership chair) Items CRUD, photos, kits, CSV import, users, reports, ledger filters, status changes (sold/disposed/maintenance/not for loan) without witness. Loan/return witness bypass (override) allowed and logged.
Member Confirmed club members with login Search, view loanable inventory, loan/return with witness, act as witness, my loans, pending witness queue, help. No catalog edit, no user admin, no reports.

No public/guest view. No self-service signup. Accounts created by Admin+ only.

3. Identity & privacy

  • Store callsign only (plus password hash, role, active flag). No addresses, phone, or other PII.
  • Personal membership data stays in the club membership database.
  • Callsigns are public by law in Canada; ledger may show callsigns freely.

4. Verification (Witness)

  • UI term: Witness
  • Hybrid: same-device co-sign is primary; remote in-app request is available
  • Witness ≠ borrower/returner; any active member with login may witness
  • Remote pending requests expire after 48 hours
  • Admin+ may bypass witness on loan/return (option A); ledger records override
  • Admin+ does not need witness for inventory maintenance (add/edit, sold, disposed, maintenance, not for loan, etc.)

Same-device loan/return

  1. Open item → Loan or Return
  2. For kits: complete include checklist + required confirmation checkbox
  3. Witness enters callsign + password on same device
  4. Confirm → status updates; ledger row written

Remote witness

  1. Start loan/return → Ask witness → pick callsign
  2. Witness sees Pending for you
  3. Confirm or decline on their device
  4. On confirm → complete; on expiry/decline → no status change

5. Items & statuses

Identifiers

  • System item ID — unique, auto-generated (required)
  • Club ID — optional free-text field (rarely used)
  • No QR codes in this build

Statuses

Available · On loan · Maintenance · Sold · Disposed

Flags

  • Not for loan — inventoried fixed/station gear (repeaters, tower, servers, etc.). Searchable; Loan hidden.
  • Photos optional (not mandatory)
  • No due dates on loans
  • No money tracking on sales — note only (e.g. flea / member callsign)

Visibility

  • Member loan/browse views: hide Sold and Disposed
  • Admin+: can still view sold/disposed items and their history

Conflicts

Unlikely; first successful save wins. Second user sees current status (e.g. already on loan).

6. Kits / Go-Kits

  • Kit = parent item with includes list (text lines and/or linked child items)
  • Loan and return both require a checklist confirmation checkbox: “I verified all included items are present” — must be checked before proceed
  • Witness still confirms the overall transaction
  • While kit is on loan, linked child items cannot be loaned separately

7. Ledger & accountability

  • Append-only ledger for loans, returns, status changes, metadata edits, admin overrides, imports
  • Field edits store old → new, who, when
  • No hard deletes: deactivate users; mark items sold/disposed; never erase ledger rows
  • Online only — no offline mode

8. Screens

  1. Login
  2. Home / Search
  3. Search results
  4. Item detail
  5. Loan (witness + kit confirm)
  6. Return (witness + kit confirm)
  7. My loans
  8. Pending witness
  9. Add/Edit item (Admin+)
  10. Mark sold/disposed/maintenance/not for loan (Admin+)
  11. Ledger
  12. Members / accounts (Admin+)
  13. CSV import + template download (Admin+)
  14. Reports (Admin+)
  15. Help (? per screen + full help page)
  16. Settings (Superuser)
  17. Security alerts / lockouts (Admin+)

Phone-first: prefer more short screens over long scrolling forms. Help uses in-page ? panels, not hover tooltips.

9. Import / export

  • Format: CSV (LibreOffice-friendly open format)
  • Admin+ can always download import template and bulk import
  • Reports downloadable as CSV

10. Reports (Admin+ only)

  1. Currently on loan
  2. Full inventory by status (incl. Not for loan filter)
  3. Aging: loanable items with no loan in the last 12 months (includes never loaned)
  4. Sold / disposed (date range)
  5. Monthly / AGM summary — large on-screen presentation view + CSV

11. Security & hosting

  • Public internet app associated with halifax-arc.org (HTTPS already in place / auto-renew)
  • Passwords hashed; DB not publicly exposed
  • Account lockout after 3 failed login attempts
  • Notify Admin+ of lockouts (in-app security alerts; email if SMTP available on host)
  • Login rate limiting; Admin+ for import/user/report actions
  • Regular DB backups on hosting/VM

12. Explicitly out of scope (this build)

  • QR codes / barcode scanning
  • Due dates / overdue fines
  • Money / payment tracking
  • Offline use
  • Training/demo mode
  • Self-service registration or guest catalog
